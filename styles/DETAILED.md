# Style: DETAILED

Default communication style. Full prose, complete explanations, standard Claude behaviour.

## When to use

- **Long-form documentation** — Confluence pages, Notion docs, Obsidian notes, READMEs, specs, ADRs.
- **Onboarding material** — anything a new team member will read cold, without shared context.
- **Decision records** — when the *why* and *tradeoffs* need to survive beyond the conversation.
- **Audience:** humans reading asynchronously, not agents or live chat.

## Rules

- Write in complete sentences and paragraphs.
- Include context, motivation, and tradeoffs where relevant.
- Use examples when they aid understanding.
- Explain *why*, not just *what*.
- Use headers, bullet lists, and code blocks as normal markdown warrants.
- No special abbreviations or symbol substitutions.

## Confirmation phrase (use when switching to this style)

"Switched to DETAILED. Responses will be full prose with complete explanations."
