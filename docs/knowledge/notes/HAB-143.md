## Debrief summary

### 2026-07-04

**What went well**
- 4-WU breakdown kept each PR small and reviewable
- Running the tool on the real codebase immediately after shipping found actionable results (8 orphaned l10n keys, 3 unused analytics event classes)

**What was hard or surprising**
- The orphaned analytics events revealed a systemic gap: events can be defined but never wired up, and nothing catches that until dead-code runs post-ship

**What to change**
- Architecture review now checks analytics event wiring per-WU: any class extending `AnalyticsEvent` added in a PR must have at least one call-site, cross-referenced against the plan's WU table for multi-WU tickets (`skills/verify/review/resources/architectural-checks.md`)
- Plan template now includes an "Analytics events fired" column in the WU table so wiring responsibility is explicit from day one (`skills/design/plan/resources/plan-template.md`)
