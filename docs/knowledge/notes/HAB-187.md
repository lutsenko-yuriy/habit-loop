### 2026-07-21 тАФ idea: CI/lint check for magic spacing numbers

While reviewing WU8 (PR #306), user asked whether a lint/CI check could catch raw numeric literals used in `EdgeInsets`/`SizedBox`/`Padding` instead of `AppSpacing` tokens, without rewriting the widget system. Out of scope for HAB-187 (token adoption is a one-time sweep, not an enforcement mechanism) тАФ two options discussed:

1. **Lightweight regex/text-scan script** (recommended starting point) тАФ mirrors `scripts/changelog/lint.py`'s existing pattern: scan `lib/` for `EdgeInsets`/`SizedBox`/`Padding` constructor calls containing a bare numeric literal, fail CI unless allowlisted (e.g. the `showup_status_dots.dart:62` exception). No new dependency, plugs into the existing `test.yml`/`run-tests` composite.
2. **`custom_lint` package** тАФ real AST-aware analyzer plugin rule, no false positives from strings/comments, shows inline in IDE like a normal lint. Repo currently only has `flutter_lints`, not `custom_lint` тАФ bigger one-time setup cost.

Filed as a follow-up ticket rather than folded into HAB-187 тАФ see the ticket for tracking.
