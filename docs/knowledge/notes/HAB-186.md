# HAB-186: Automate session-start checklist via SessionStart hook

## Notes

### 2026-07-20

**Reply to audit-code comments inline, not as standalone PR comments**

When responding to `[audit]`/`[review]` findings (fix or explain-why-not), post the reply as a threaded reply under the specific review comment, not as a new top-level PR comment. Standalone comments make it hard for the user to tell which reply answers which finding. Use `gh api repos/{owner}/{repo}/pulls/{pr}/comments/{comment_id}/replies` (or the equivalent inline-reply endpoint) instead of `gh pr comment`.
