# HAB-163: Too-many-pacts dialog copy implies a hard limit, but it's a soft warning

## Notes

### 2026-07-13

**Splitting mixed briefs by workflow type, not just by ticket**

User opened the session with "let's brief" for two items, but item 1 was actually a bug (misleading dialog copy) and item 2 was a research question (finding an on-the-fly l10n update service). Instead of forcing both through the feature-brief dialog just because the session started with "brief", flagged that they fit `docs/TROUBLESHOOT_WORKFLOW.md` and `docs/RESEARCH_WORKFLOW.md` respectively and asked the user to confirm the split before proceeding. User explicitly liked this — confirms the instinct to route by *what the item actually is* rather than by the literal invocation word should carry forward.
