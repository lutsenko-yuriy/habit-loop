# HAB-161: Give skill_router's script stub an explicit cheap fallback alias

## Notes

### 2026-07-12

**Planning surfaced a bigger win than the ticket framed**

The `plan` pass found that 6 of 11 script-stub commands (`android`, `ios`, `debrief`, `run-scenarios`, `skill-creator`, `style`) declare `needs_session_tools: true`, which means they always exit non-zero and always hit the uncapped fallback today — they never even attempt LM Studio. The ticket named only 3 commands (`implement`, `style`, `summarize`), but the real-world exposure to the uncontrolled-cost fallback is much larger. Scope was expanded to all 11 script-stub commands as a result. Worth naming at debrief: this is a case where letting the planning pass actually read the current `.claude/commands/*.md` state (rather than trusting the ticket's own estimate) caught a materially bigger problem than the ticket author knew about at filing time.
