# HAB-177: Simplify PactTimelineGrouper — remove dead grouping-threshold complexity

## Notes

### 2026-07-17 — WU scope summary (proposed plan, pending approval)

Pure refactor, no real-user-visible change (`groupingThreshold` has been `1` in every production build, so the group-milestone machinery is dead code). Full plan text: see the `/plan` comment on the HAB-177 Linear issue.

- **WU0** — integration scenarios only, no production code (`[test]`, merge directly). Rewrites the `short_mixed_run_shown_as_group_item` scenario in `pact_timeline_flow_test.dart` to assert the mixed short run renders as separate streak milestones (the real production threshold-1 behaviour), removing its artificial threshold-10 override. Green on current `main` — it locks in existing behaviour rather than introducing new behaviour.
- **WU1** — grouper single-pass rewrite + config/RC removal (`[wip]`). Rewrites `PactTimelineGrouper.group()` as one fused forward pass (tail-bucketing + non-tail streaks), deletes the group accumulators / `flushGroup` / threshold branches, removes the `groupingThreshold` constructor param, `PactTimelineConfig.milestoneGroupingThreshold`, and the `pact_timeline_milestone_grouping_threshold` RC default/`all`/`intRanges` entries. Keeps `ShowupGroupMilestone` **defined** (grouper just stops emitting it) so UI + l10n still compile between WUs. Updates the grouper/config/RC-defaults/cache/view-model/service tests + the `_rcGrouping` integration helper. ~11 files (1 over the soft cap — the helper is tied to the RC-key removal).
- **WU2** — remove `ShowupGroupMilestone` type + UI + l10n (`[app]` — final WU, builds, but no user-visible change so no release note). Deletes the sealed variant, all exhaustive-`switch` arms (formatters, spine, iOS/Android pages, `_GroupLabel`), the `timelineGroup` l10n key across 4 arb files, and the covering tests; updates `docs/ARCHITECTURE.md`. Intentionally a single atomic WU exceeding the 10-file soft cap: removing a `sealed` variant forces its definition, every consumer, its l10n key, and its tests to change together — splitting would leave a non-compiling intermediate WU. Net-negative LoC (nearly all deletions).

**Sequencing:** land HAB-177 fully before HAB-174 WU2/WU3 (WU1 changes the grouper constructor shape; both tickets edit `pact_detail_cache_test.dart`'s `_describe`/golden fixtures). HAB-174's planned WU1.1 (revisit `PactDetailCache._computeBundle` single-pass fusion) depends on this simplification landing first.
