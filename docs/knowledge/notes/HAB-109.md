### 2026-07-04

**WU1 — What a software licence does**

A licence grants (or withholds) the right to use, copy, modify, and distribute the software. Without an explicit licence, copyright law applies by default: all rights reserved, no reuse permitted. This applies even to public repositories — pushing code to GitHub does not grant anyone the right to reuse it.

Licence families:

| Family | Examples | Obligations for downstream users |
|---|---|---|
| Public-domain | CC0, Unlicense | None |
| Permissive | MIT, BSD-2, BSD-3 | Retain copyright notice and licence text |
| Permissive + patent | Apache 2.0 | Same as above; also includes explicit patent grant and retaliation clause |
| Weak copyleft | LGPL 2.1/3, MPL 2.0 | Modifications to *the library itself* must be released under the same licence |
| Strong copyleft | GPL v2, GPL v3 | Entire derivative work must be released under the same licence |
| Virulent copyleft | AGPL v3 | Same as GPL, plus network use triggers copyleft |
| Proprietary | "All rights reserved" | No reuse; controlled by the copyright holder |

Source code licence and App Store distribution are independent — a MIT-licensed codebase can be compiled and sold as a proprietary app on the App Store with no conflict.

**WU2 — Recommendation**

Project context as of June 2026: solo developer, no contributors planned, public GitHub repo, no commercial plans, primary use case is sharing with colleagues. Codebase is largely AI-generated.

AI-generated code and copyright: copyright requires human authorship (US Copyright Office). Code where a human provided architectural direction, domain modelling, prompt design, review, and approval likely retains *some* human authorship — but how much is an open question. Anthropic's Consumer Terms of Service grant the user ownership of Claude's output (subject to applicable law). Asserting strong proprietary rights over AI-generated code is legally shakier than for hand-written code; applying a permissive open-source licence is more honest.

Candidates evaluated:

| Licence | Verdict |
|---|---|
| **MIT** | **Recommended** — shortest text, universally understood, compatible with all deps, aligns with studying/sharing goal |
| Apache 2.0 | Strong second choice — adds patent grant; NOTICE file requirement is extra overhead |
| GPL v2 | **Ruled out** — incompatible with Apache-2.0 transitive deps (`fake_async`, `clock`, `material_color_utilities`) |
| Proprietary / no licence | Not recommended — legally prevents colleagues from reusing code; shakier for AI-generated work |

MIT vs. no licence: current "no licence" state technically prevents colleagues from studying or sharing the code even though the repo is public. MIT resolves this with one file and zero ongoing obligations for the author.

Decision was deferred at research time due to mild AI copyright uncertainty and no external pressure. Trigger condition applied on 2026-07-04: "simply a good moment with no other priorities."

**WU3 — Dependency compatibility audit**

All direct `pub.dev` dependencies are MIT, BSD-2-Clause, BSD-3-Clause, or Apache-2.0 — all permissive, all compatible with MIT. No copyleft library present. No third-party source was found copy-pasted into `lib/` (scanned for copyright notices — zero hits). Full compatibility table was verified for all 23 direct deps and 8 notable transitive deps.
