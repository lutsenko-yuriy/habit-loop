# HAB-166: Onboard first iOS tester to Firebase App Distribution

## Notes

## Debrief summary

### 2026-07-14

**What went well**
- The onboarding steps (Firebase group, UDID registration, provisioning-profile regeneration, secret update, CI dispatch) worked as documented on the ticket — first iOS tester installed successfully once the refreshed profile was distributed.

**What was hard or surprising**
- Adding an iOS tester to Firebase App Distribution takes disproportionate effort: UDID lookup, Apple Developer portal device registration, provisioning-profile regeneration, and a GitHub secret update — all for one tester. This is a structural FAD/ad-hoc limitation, not a gap in how this ticket was executed.

**What to change**
- Nothing to change in this workflow. The root cause (ad-hoc profile UDID requirement) is already tracked separately in HAB-167 (TestFlight distribution), which removes the UDID pre-registration step entirely.
