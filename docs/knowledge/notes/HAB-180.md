## Debrief summary

### 2026-07-18

**What went well**
- Near-perfect ticket overall — CI graph rewiring (job removal, OR-gate restore for `version-tag`) went smoothly.

**What was hard or surprising**
- Apple's App Store Connect build processing is the main friction point: it's async and can take minutes to tens of minutes, and TestFlight's "What's New" notes can only be uploaded once the build has finished processing and is visible via the App Store Connect API — `altool` alone can't set it. Tracked as HAB-182.

**What to change**
- Want manually-dispatched, granular CI pipelines (build-only, test-only, notes-generate-and-upload, etc.) for both platforms, in new workflow file(s) separate from the already-large `ci.yml` — to iterate on a single procedure without paying for the full pipeline. `ci.yml` itself likely needs renaming once responsibilities split across files. Tracked as HAB-183.
- `/dead-code-check`'s orphaned-test-file detector produced 3 false positives this run (`test_runner.dart`, `language_change_flow_test.dart`, `onboarding_carousel_flow_test.dart`) — it only checks for direct `package:habit_loop/` imports and misses files linked via a shared test harness, relative imports between test files, or direct path references in CI/skills. Tracked as HAB-184.
