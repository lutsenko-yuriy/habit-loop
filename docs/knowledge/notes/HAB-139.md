## Notes

- 2026-06-30: if after finishing a WU some scenarios should go green, the implementation should be finished with filling these scenarios.
- 2026-06-30: when filling scenario stubs, cover all steps from the original TODO list — do not silently drop tail steps (e.g. "navigate back and verify stats", "verify event parameters in full"). Either implement them or explicitly justify why they are skipped in a code comment.

- 2026-06-30: feature-flag access must go through `featureFlagsProvider` (i.e. `ref.read(featureFlagsProvider).someFlag`), not directly through `remoteConfigServiceProvider.getBool(...)`. Bypassing `FeatureFlags` causes drift: source/default changes in `FeatureFlags.fromRemoteConfig` won't propagate. Raw RC reads are fine for configuration values (integers like tail period), but toggle booleans belong in `FeatureFlags`.
- 2026-06-30: when adding a new action button, check that `isSaving` is guarded at the call site (pass `null` to `onPressed` while saving). `markDone`/`markFailed` both do this; the redemption button was missed. Without the guard a fast double-tap can fire two concurrent `persistShowupStatus` calls with redundant DB writes and duplicate analytics events.
- 2026-06-30: after a successful redemption (or any status change), clear all transient UI flags that become stale — including `wasAutoFailed`. Leaving it `true` after redemption keeps the auto-fail error banner visible alongside the "Done" badge.
- 2026-06-30: `[user]` CHANGELOG bullets must be written in plain user language with no internal references — no ticket numbers, WU prefixes, or implementation terms. Readers of "What's New" are end users.
- 2026-06-30: prefer a single-pass partition loop over two separate `where` passes when splitting a list into two disjoint groups; it is more readable and avoids iterating the list twice.
- 2026-06-30: be aware of dashboard gap-fill: on startup the dashboard auto-fails any newly generated showups whose `scheduledAt` is before today. Integration tests that seed a showup and check `showupsFailed` after an operation must account for this — or place the showup on the day immediately before today so the gap window is empty.
